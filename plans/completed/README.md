# Completed Plans

This folder archives feature plans that are already implemented and validated.

- For the current repo state, next recommendation, and parallel-safe work, read `plans/STATE.md` first.
- Broad unfinished work stays in `plans/epics/` until it is refined.
- Active and implementation-ready child plans stay in `plans/`.
- Completed feature plans move to `plans/completed/{feature_name}/`.
- Use archived plans as read-only history for prerequisite context, prior decisions, and testing artifacts.

Do not use this folder README to decide what is next. `plans/STATE.md` is the durable source of truth for active, blocked, next, parallel, and recently archived work.
