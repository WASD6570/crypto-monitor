# Go Services

This directory contains focused Go service boundaries for the live and data-intensive path.

Each service folder is intentionally shallow at the start so later implementers can add internals without fighting a premature structure.

- `venue-binance`
- `venue-coinbase`
- `venue-bybit`
- `venue-kraken`

- `feature-engine`
- `regime-engine`
- `alert-engine`

- `replay-engine`
- `risk-engine`
- `simulation-api`
- `backfill-engine`
